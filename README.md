# My Home Clone

ブラウザで動く、完全ローカル完結型の **3D間取り・家具配置シミュレーター**。
注文住宅の自己検討ツール。将来的に **日射・日当たりシミュレーション** を独自機能として追加予定。

> 「マイホームクラウド」「住宅スキャン」は他社登録商標。本プロジェクトは個人利用で、名称は **My Home Clone / マイホームクローン** を用います。商用配布はしません。

## 技術構成

- 純粋なフロントエンドのみ。**ビルドツール・バンドラ不要**（Vite/webpack なし）。
- ES Modules（`type="module"`）でファイル分割。
- 3D は **Three.js** を CDN + importmap で読み込み（npm 不要）。
- 永続化は **LocalStorage 自動保存** ＋ **JSON エクスポート/インポート**。
- 外部送信・サーバ通信は一切なし（完全ローカル）。Three.js の CDN 取得のみネット接続が必要。

## 起動方法

VS Code / Cursor の **Live Server**（Port 5500 想定）で `index.html` を開くだけ。
または任意の静的サーバで配信:

```bash
python3 -m http.server 5500
# → http://localhost:5500/
```

※ `file://` 直開きは ES Modules / importmap の制約で動かないため、必ずローカルサーバ経由で開いてください。

## ファイル構成

```
index.html        画面骨格 + importmap
css/style.css     ダークモードUI
js/main.js        エントリ。UI配線・統括
js/store.js       状態管理・LocalStorage・JSON入出力・プラン管理
js/model.js       データ構造／座標変換／面積・畳の計算（純粋関数）

js/viewer3d.js    Three.js の 3D 描画
js/sun.js         日射計算（フェーズBで実装。現状は空）
```

## 座標・単位の規約

- 内部単位は **mm**。`X`=間口（左右）、`Z`=奥行（上下）、3D の `Y`=高さ。Three.js の Y-up 右手系に一致。
- 1マス = **1P = 910mm**。スナップ = `910 / snapDivisions`（既定 4 = 0.25P / UIで 0.5P・0.25P 切替）。
- 面積はシューレース公式で算出し m² に換算。畳数 = 面積(m²) ÷ `meta.tatamiM2`（既定 1.62）。
- 図面表示は「12.4㎡（7.6帖）」形式。

## 使い方（フェーズA）

1. 上部で **2D / 3D** とフロア **1F / 2F / 3F** を切替。
2. 左の **ツール**で操作を選択:
   - **部屋を描く**: 種別を選び、作図エリアでドラッグして矩形の部屋を作成（壁は外周に自動生成）。
   - **家具を置く**: 家具を選びクリックで配置。
   - **選択 / 移動**: 部屋・家具をクリックで選択、ドラッグで移動。`Del` 削除、`R` 家具回転。
   - **画面移動**: ドラッグでパン（ホイールでズーム）。
3. 各部屋に **面積㎡と帖** を自動表示。
4. **3D** で壁を天井高まで立ち上げて立体表示（OrbitControls で視点操作）。
5. 変更は **自動で LocalStorage 保存**。**JSON書出 / 読込** とプランの **新規・複製・削除** が可能。

## ロードマップ

- **フェーズA（実装済）**: 2D作図・面積/帖・家具・3Dビュー・永続化・JSON入出力・プラン管理。
- **フェーズB**: 日射シミュレーション（太陽位置・影・直射時間の季節別集計）。板橋区赤塚 北緯35.775 / 東経139.679。
- **フェーズC**: 建具/住設/外構の充実、敷地写真の下絵トレース、壁の手動編集、寸法線、PDF/画像書出。
